# KB Remote (sony-tv-remote)

iPhone app that acts as a remote control for Sony Bravia TVs (and any Android TV — Shield, Chromecast with Google TV) over local WiFi. No cloud, no account. Phone talks directly to TV using the Android TV Remote Service v2 protocol.

The full user-facing README is in [README.md](README.md) — read it for setup, pairing flow, and protocol details before doing anything protocol-related.

## Stack

- **Expo SDK 54** + **React Native 0.81.5** + **React 19.1** + **TypeScript 5.3**
- **Expo Dev Client** (no Expo Go — we use a custom native module)
- Navigation: `@react-navigation/native-stack` v7
- Storage: `expo-secure-store` (cert, key, IP)
- Crypto: `node-forge` for X.509 client cert generation in pure JS
- Wire format: `protobufjs` + hand-rolled varint length prefix framing
- Native iOS module: `modules/atv-cert/` (Swift, Network.framework mTLS)

## Architecture (read before touching protocol code)

```
src/
├── polyfills.ts                  Buffer global — required for node-forge & protobuf framing
├── api/
│   ├── keycodes.ts               Android KeyEvent codes (POWER=26, DPAD_*, etc.)
│   ├── atvCert.ts                X.509 client cert generation via node-forge (2048-bit RSA)
│   ├── atvProto.ts               Protobuf schemas + varint framing (pairing + remote)
│   └── atvClient.ts              Pairing client (port 6467) + Remote client (port 6466)
├── components/                   RemoteButton (haptics/debounce/hold-repeat), DPad, Rocker, NumberPad, Toast
├── context/SettingsContext.tsx   Persists IP + client cert (SecureStore)
├── screens/                      RemoteScreen (main), SettingsScreen, PairingScreen
└── theme/colors.ts

modules/atv-cert/                 Native module
├── index.ts                      JS-side facade (AtvCert + AtvTlsConnection class)
└── ios/                          Swift sources — uses Network.framework + Keychain
```

`App.tsx` wires: `SafeAreaProvider > SettingsProvider > ToastProvider > NavigationContainer`.

## Protocol invariants (DO NOT change without checking)

- **Port 6467** = pairing (one-time, user types 6-char code from TV).
- **Port 6466** = remote control (mTLS with paired cert).
- Each TLS payload = single protobuf message, single-varint length-prefixed.
- Protobuf field numbers in remote service must match `tronikos/androidtvremote2` and `louis49/androidtv-remote` — see the table in `README.md`. Mismatch → `RemoteError value: 5`.
- Pairing secret = `SHA-256(client_pubkey + server_pubkey + code_nonce)`. The hex code shown on TV encodes both the nonce *and* a checksum of it.

## Conventions

- File naming: `camelCase.ts` for non-component modules, `PascalCase.tsx` for components/screens.
- `react-native` imports use the bare-workflow style (we are NOT on Expo Go).
- All network certs/keys live in **SecureStore**, never AsyncStorage.
- Hot-reload friendly: JS edits work via `npx expo start --dev-client`. Swift/native edits need a fresh Xcode build.

## Commands

```bash
npm start           # Metro dev server (dev client)
npm run typecheck   # tsc --noEmit
npm run ios         # native iOS build via expo run:ios
npx expo prebuild --platform ios --clean   # regenerate ios/ folder (not in git)
```

The `ios/` folder is **gitignored** — it's generated by `expo prebuild`. Don't hand-edit; edits get blown away. Native iOS work goes in `modules/atv-cert/ios/`.

## Gotchas you'll hit

- **"TV closed pairing connection"** = wrong cert/key wired into TLS — almost always a Buffer polyfill issue. Rebuild the dev client.
- **`pod install` says "No such Pod: AtvCert"** = `expo prebuild` was skipped. Re-run it.
- **Cert generation feels stuck** = node-forge does 2048-bit RSA in pure JS; up to a minute on older devices is normal.
- **Free Apple ID** = app expires every 7 days, requires re-run from Xcode.

## What NOT to do

- Don't commit `ios/` (gitignored — regenerated).
- Don't store secrets in AsyncStorage — use `expo-secure-store`.
- Don't change protobuf field numbers without cross-checking the reference implementations.
- Don't switch to Expo Go — the native `atv-cert` module won't work there.
